# PEAS（智能体任务环境描述框架）

## 一句话解释

PEAS 是一个用来精确描述"智能体在什么环境下做什么事"的分析框架，包含四个维度：**P**erformance（性能度量）、**E**nvironment（环境）、**A**ctuators（执行器）、**S**ensors（传感器）。

## 它解决什么问题？

在设计智能体时，必须先搞清楚"智能体要面对的世界是什么样的"。PEAS 提供了一套结构化的问题模板，帮助设计师系统性地定义任务环境，避免遗漏关键约束。

## 为什么重要？

PEAS 是 Agent 设计的第一步。如果连"智能体的目标是什么？它能感知什么？它能做什么？环境有什么特性？"都没想清楚，后面的实现一定会出问题。

## 四个维度

| 维度 | 含义 | 问题 |
|------|------|------|
| **P**erformance | 如何衡量成功 | 怎样算做得好？ |
| **E**nvironment | 智能体所处的世界 | 环境有什么特性？ |
| **A**ctuators | 智能体影响环境的方式 | 能做哪些操作？ |
| **S**ensors | 智能体获取信息的方式 | 能感知什么？ |

## 使用场景

- **设计新 Agent**：用 PEAS 明确边界和约束
- **分析已有 Agent**：用 PEAS 理解其设计假设
- **对比不同方案**：同样的任务，不同 Agent 的 PEAS 可能不同

## 实例：旅行助手的 PEAS

| 维度 | 描述 |
|------|------|
| Performance | 推荐准确度、响应速度、用户满意度 |
| Environment | 互联网、旅行数据库、实时 API |
| Actuators | 天气查询 API、景点搜索 API、文本回复 |
| Sensors | 用户输入、API 返回数据 |

## 容易误解的点

- **PEAS 不是架构设计**：它描述"要解决什么问题"，不描述"怎么实现"
- **环境特性比工具更重要**：环境的可观察性、确定性、动态性直接影响 Agent 的设计策略

## 和其他概念的关系

- [[Agent]]：PEAS 是定义 Agent 任务环境的工具
- [[Environment]]：PEAS 中的 E 是整个框架的核心约束

## 我的例子

设计一个"会议纪要助手"时，用 PEAS 分析：
- **P**：纪要准确率、生成速度、关键决策捕获率
- **E**：飞书/钉钉会议、音频流、参会人列表
- **A**：写入文档、标记待办、@相关人员
- **S**：麦克风输入、日历事件 API、参会人信息
