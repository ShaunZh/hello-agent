# Memory（智能体记忆）

## 一句话解释

Memory 是让 **无状态的 LLM** 能记住对话历史、用户偏好和交互经历的系统，它模拟人类记忆的层次结构：感觉→工作→短期→长期。

## 它解决什么问题？

LLM 每次 API 调用都是**独立的**，不会自动记住上一轮聊了什么。Memory 系统通过外部存储（内存、数据库、向量库）来记录和检索这些信息，解决：
- 长对话中上下文丢失
- 无法记住用户的偏好和习惯
- 无法从历史经验中学习改进
- 多轮对话中出现前后矛盾

## 四种记忆类型（对照人类记忆）

| 类型 | 人类对应 | 存储 | 生命周期 | 用途 |
|------|---------|------|---------|------|
| **WorkingMemory** | 工作记忆 | 纯内存 | 单会话，TTL 自动清理 | 当前对话上下文 |
| **EpisodicMemory** | 情景记忆 | SQLite+Qdrant | 长期 | 具体事件（"昨天问了Python问题"） |
| **SemanticMemory** | 语义记忆 | Neo4j+Qdrant | 持久 | 抽象知识（"Python是面向对象的"） |
| **PerceptualMemory** | 感知记忆 | SQLite+Qdrant | 按重要性动态管理 | 多模态（图片、音频） |

## 核心操作

| 操作 | 含义 |
|------|------|
| `add` | 添加记忆（指定类型和重要性） |
| `search` | 语义检索记忆 |
| `forget` | 遗忘（按重要性/时间/容量） |
| `consolidate` | 整合：短期→长期 |
| `summary` | 获取记忆摘要 |

## 评分算法（各类型不同）

```
工作记忆： (TF-IDF相似度×0.7 + 关键词×0.3) × 时间衰减 × (0.8+重要性×0.4)
情景记忆： (向量相似度×0.8 + 时间近因性×0.2) × (0.8+重要性×0.4)
语义记忆： (向量相似度×0.7 + 图相似度×0.3) × (0.8+重要性×0.4)
```

情景记忆更强调"时间近因性"（最近的事更重要），语义记忆更强调"图关系"（概念间的关联）。

## 容易误解的点

- **Memory ≠ 把对话历史全塞进 Prompt**：那会撑爆上下文窗口。真正的 Memory 要有检索、有遗忘、有整合。
- **不是所有记忆都同等重要**：`importance` 参数决定了记忆在检索中的权重，以及对遗忘的抵抗力。
