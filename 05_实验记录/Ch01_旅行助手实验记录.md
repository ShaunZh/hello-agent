# Ch01 旅行助手实验记录

## 1. 实验目标

运行 Ch01 1.3 节的智能旅行助手代码，验证 Thought-Action-Observation 循环的实际工作流程，理解 LLM Agent 如何调用外部工具。

## 2. 运行环境

- 操作系统：macOS
- Python 版本：待确认
- 依赖安装：`pip install requests tavily-python openai`

## 5. 预期输出

```
用户输入: 你好，请帮我查询一下今天北京的天气，然后根据天气推荐一个合适的旅游景点。
========================================
--- 循环 1 ---
Thought: 首先需要获取北京今天的天气情况...
Action: get_weather(city="北京")
Observation: 北京当前天气:Sunny，气温26摄氏度
--- 循环 2 ---
Thought: 天气晴朗，接下来推荐景点...
Action: get_attraction(city="北京", weather="Sunny")
Observation: 北京在晴天最值得去的旅游景点是颐和园...
--- 循环 3 ---
Thought: 信息充足，可以回答用户...
Action: Finish[今天北京的天气是晴朗的...]
```

## 6. 我的理解

这个实验展示了 TAO 循环的三个关键特征：

1. **LLM 即"大脑"**：模型在 Thought 阶段自己规划"先查天气→再推荐景点"的两步流程，不需要人工写 if-else
2. **工具即"手脚"**：Action 让 LLM 突破"只能生成文本"的限制，可以获取实时数据
3. **循环即"工作方式"**：不是一次调用完成所有事，而是逐步推进，每步都基于上一步的结果
